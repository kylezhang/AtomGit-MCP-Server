你必须严格基于该文件重构当前项目 `@src` 中的 MCP Tool 实现，**禁止任何形式的自由发挥或主观判断**。

### 强制执行规则（不可违反）

1. **接口与工具数量**

   * 必须实现 **且仅实现 240 个 MCP Tool**。
   * 【文件】中的每一个接口 **必须且只能**对应一个 MCP Tool。
   * 当前代码中任何 **不在【文件】240 个接口内**的实现，**全部删除**。

2. **分类来源与数量**

   * 接口分类 **只能来自【文件】定义的 18 个分类**。
   * **禁止新增、合并、拆分、重命名分类**。

3. **`@src/tools` 目录（最高优先级）**

   * 目录下 **只能存在 18 个 `.ts` 文件**，**不多不少**。
   * 每个 `.ts` 文件 **必须一一对应【文件】中的一个分类**。
   * 每个文件中包含的 MCP Tool：

     * **只能属于该分类**
     * **接口数量与【文件】完全一致**
   * 当前存在的 **22 个工具文件属于错误状态，必须调整为 18 个**。

4. **Service 层**

   * `@src/services/AtomGitService.ts` 中 **必须存在 240 个 `async` 方法**。
   * 每个方法 **一一对应【文件】中的一个接口**，不得跨分类、不得复用。

5. **归属一致性**

   * 分类 → 工具文件 → MCP Tool → Service 方法 → API 接口
     必须 **完全一一对应**。
   * **任何不一致均视为任务失败**。
