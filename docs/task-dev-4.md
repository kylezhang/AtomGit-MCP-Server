## System Prompt（接口数量一致性与工具语义校验 · 强约束）

### 权威事实（不可争议）

* `@docs/api_endpoints_extracted.json` **仅定义 240 个接口**，不存在第 241 个或更多接口。
* 因此：

  * **最终 MCP Tool 数量必须且只能是 240 个**
  * 任何 **246+、241+** 的结果 **一律视为错误**

---

### 当前状态确认

* 基于 `@docs/api_endpoints_extracted.json` 的 **18 个分类**：

  * `@src/tools` 的拆分结构 **正确**
  * `@src/services` 的拆分结构 **正确**
* 但当前实现中，**MCP Tool 总数 ≠ 240**，存在数量偏差，必须进一步核查与修正。

---

### 强制任务（不可跳过）

1. **数量差异溯源**

   * 逐一对比：

     * `@docs/api_endpoints_extracted.json` 中的 240 个接口
     * 当前已实现的 MCP Tools
   * 明确找出：

     * 多出来的 Tool 来自哪里
     * 是重复实现、误拆分，还是错误新增

2. **严格对齐与修正**

   * 对不在 240 接口清单内的 MCP Tool：

     * **全部删除**
   * 对因拆分、命名或归属错误导致的重复 Tool：

     * **合并或修正为唯一实现**
   * 修正后：

     * MCP Tool 总数 **必须回到 240**

3. **工具命名与描述校验（强制）**

   * 对 **全部 240 个 MCP Tool**：

     * 校验 Tool 名称是否准确反映其接口语义
     * 校验 Tool 描述是否：

       * 与接口真实能力一致
       * 无歧义、无泛化、无误导
       * 有助于大模型进行正确的语义理解与调用
   * 禁止：

     * 描述与接口能力不匹配
     * 多个 Tool 使用高度相似但不可区分的描述

4. **权威对照规则**

   * Tool 名称与描述：

     * 必须以 `@docs/api_endpoints_extracted.json` 为主
     * 如存在疑问，**必须**通过其中提供的 API URL 访问官方文档确认
   * **严禁** 自行推断、简化或“看起来合理”的命名与描述。

---

### 禁止提前停止（强制）

* 在以下任一条件未满足前，任务一律视为 **未完成**：

  * MCP Tool 总数 ≠ **240**
  * 任一 Tool 不在权威接口清单中
  * 任一 Tool 名称或描述存在语义不准确问题
  * 未查清并修正“多出接口工具”的根因

**不得输出“已完成”“结构已正确”等结论，除非全部条件同时满足。**

---

### 最终验收标准（必须全部满足）

* 接口分类数：**18**
* MCP Tool 总数：**240**
* 每个 Tool ↔ 一个接口 ↔ 一个分类
* Tool 名称准确、唯一、无歧义
* Tool 描述准确反映接口能力，利于大模型理解
* 不存在任何多余、重复或臆造的接口工具
